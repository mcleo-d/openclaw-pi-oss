# Support

This is a hobby project. Support is best-effort with no SLA.
Read the relevant documentation before opening an issue — it covers most deployment questions.

---

## I have a deployment or configuration question

Read [`config/README.md`](config/README.md) and
[`docs/04-docker-openclaw.md`](docs/04-docker-openclaw.md) first.
These cover the full deployment sequence, all proxy environment variables, and common
troubleshooting scenarios.

If your question is not answered:

[Open a Config Question issue](https://github.com/mcleo-d/openclaw-pi-oss/issues/new?template=config_question.md)

Include your Pi model, OS version, Ollama version, and what you have already tried.
Remove all real values (tokens, hostnames, IP addresses) before posting.

---

## I found a bug

[Open a Bug Report](https://github.com/mcleo-d/openclaw-pi-oss/issues/new?template=bug_report.md)

Include the environment table from the template and relevant logs. These are essential
for diagnosis — issues without them cannot be triaged effectively.

---

## I have a security concern

**Do not open a public issue.**

See [SECURITY.md](SECURITY.md) for the private disclosure process.
Security issues reported publicly will be closed and the reporter directed here.

---

## I want to suggest a feature or roadmap item

Check [ROADMAP.md](ROADMAP.md) first — your idea may already be tracked.

[Open a Feature Request](https://github.com/mcleo-d/openclaw-pi-oss/issues/new?template=feature_request.md)

---

## I want to contribute

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full workflow.

---

## Response times

| Type | Expectation |
|---|---|
| Security reports | Acknowledged within 7 days (SECURITY.md) |
| Bug reports | Triaged within 14 days, best effort |
| Config questions | Best effort, 14–30 days typical |
| Feature requests | Best effort, no timeline commitment |
| Pull requests | Reviewed within 7–14 days |

These are targets, not guarantees. This project has one maintainer working in their
own time. If faster support is needed, the [OpenClaw project](https://openclaw.ai)
may be a better starting point for questions about OpenClaw itself.
